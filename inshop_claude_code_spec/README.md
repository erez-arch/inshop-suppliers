# INSHOP / CYB-ORG - חבילת אפיון מלאה ל-Claude Code

חבילה זו הופכת את מסמך האפיון העסקי למסמכי עבודה ישימים לפיתוח: דרישות מוצר, UX/UI, ארכיטקטורה, מודל נתונים, API, מכונות מצבים, תכנית בדיקות, החלטות ברירת מחדל ואבטיפוס מסכים רספונסיבי.

## התחלה מהירה

- מסמך יחיד להזנה: `INSHOP_FULL_SPEC_FOR_CLAUDE_CODE.md`.
- חוזה עבודה קבוע בריפוזיטורי: `CLAUDE.md`.
- אבטיפוס אינטראקטיבי: `prototype/index.html`.
- תצוגה מקדימה של כל המסכים: `prototype/previews/contact_sheet.jpg`.

## סדר קריאה מומלץ ל-Claude Code

1. `CLAUDE.md` - חוזה העבודה והכללים שאסור להפר.
2. `docs/01_PRODUCT_SPEC_HE.md` - האפיון הפונקציונלי המלא.
3. `docs/02_UI_UX_SPEC_HE.md` - אפיון כל המסכים והמצבים.
4. `docs/03_TECHNICAL_ARCHITECTURE.md` - ארכיטקטורה, אינטגרציות, אבטחה ותפעול.
5. `docs/04_STATE_MACHINES.md` - סטטוסים, מעברים, תנאי סף ותופעות לוואי.
6. `docs/05_API_CONTRACT.yaml` - חוזה API מוצע.
7. `docs/06_DATABASE_SCHEMA.sql` - מודל נתונים מוצע ל-PostgreSQL.
8. `docs/07_TEST_PLAN_HE.md` - תכנית בדיקות וקריטריוני קבלה.
9. `docs/08_IMPLEMENTATION_PLAN_HE.md` - תכנית פיתוח לפי שלבים ו-vertical slices.
10. `docs/09_DECISIONS_AND_ASSUMPTIONS_HE.md` - החלטות ברירת מחדל ושאלות שעדיין ניתן לשנות.
11. `docs/10_TRACEABILITY_MATRIX_HE.md` - מיפוי בין דרישות המקור למסמכי החבילה.
12. `prototype/index.html` - אבטיפוס מסכים אינטראקטיבי, RTL ורספונסיבי.

## אופן שימוש מומלץ

- להעתיק את `CLAUDE.md` לשורש הריפוזיטורי.
- להשאיר את תיקיית `docs/` בתוך הריפוזיטורי ולדרוש מ-Claude Code לצטט בכל PR את סעיפי האפיון שיושמו.
- לפתוח את `prototype/index.html` בדפדפן ולהשתמש בו כ-reference ויזואלי, לא כקוד production.
- להתחיל משלב 0 בתכנית הפיתוח, ולא לבנות AI אמיתי לפני שכל המסלולים הידניים עובדים מקצה לקצה.
- כל שינוי עסקי מחייב עדכון של: מסמך המוצר, מכונת המצבים, ה-API, הסכמה ותכנית הבדיקות.

## עקרונות שאינם נתונים לפרשנות

- כל גישה למערכת הישנה עוברת רק דרך `LegacyAdapter`.
- `ExcelLegacyAdapter` הוא המימוש הראשון; `RealLegacyApiAdapter` מחליף אותו ללא שינוי בלוגיקה העסקית.
- AI מציע, מסמן ומחלץ מידע; הוא אינו מבצע עדכון מלאי, אישור זיכוי או סגירת תשלום ללא אישור משתמש מורשה.
- כל שינוי מלאי או כסף נרשם כתנועה בלתי-מחיקה עם Audit מלא.
- הממשק בעברית, RTL; מסכי ספק, נאמן וספירה הם mobile-first; מסכי אדמין ותשלומים הם desktop-first; פורטל ספק רספונסיבי מלא.
