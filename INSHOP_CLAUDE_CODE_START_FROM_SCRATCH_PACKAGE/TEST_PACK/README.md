# INSHOP Supplier Flow Test Pack

חבילת בדיקות סינתטית למערכת ספקים / נאמנים / אדמין / זיכויים / תשלומים.

חשוב: כל החשבוניות, הזיכויים ואישורי התשלום כאן הם מסמכי דמו בלבד, לצורך QA ו-OCR. הם נראים כמו מסמכים עסקיים אמיתיים אך אינם חשבוניות מס אמיתיות.

## מבנה
- `pdf/` - חשבוניות חיוב, חשבוניות זיכוי ואישורי תשלום בפורמט PDF.
- `jpg/` - אותם מסמכים כ-JPG, כדי לדמות צילום/סריקה במובייל.
- `csv/` - נתוני Mock ל-ExcelLegacyAdapter.
- `inshop_mock_data.xlsx` - אותו דאטה כאקסל רב-לשוני עם כל הגיליונות.
- `ai_expected_outputs.json` - דוגמאות לפלט OCR/AI צפוי.
- `CLAUDE_CODE_ADDENDUM.md` - הוראות השלמה לקלוד קוד.

1. ספק מדווח אספקה עם `INV-TNV-12548.jpg`.
2. נאמן פותח קישור, מאשר שבפועל התקבלו 216 יחידות חלב במקום 240.
3. אדמין רואה חוסר 24 יחידות ומייצר דרישת זיכוי 152.93 ₪.
4. ספק מעלה `CN-TNV-80125.jpg`.
5. אדמין מאשר את הזיכוי.
6. אדמין בוחר לתשלום את `INV-TNV-12547` ו-`INV-TNV-12548`, מקזז `CN-TNV-80125`, ומעלה `PAY-BANK-33001.jpg`.
7. המערכת מסיימת עם חשבוניות paid, זיכוי closed, ותיעוד Audit מלא.
